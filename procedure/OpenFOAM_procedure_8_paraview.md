# 手順書 [8] ParaViewでの処理・確認（ブランチB）

> 上位ルーティン `OpenFOAM_calc_routine.md` の §2[8]・§6 を展開した詳細手順書。
> 操作の詳細は `ParaView_cheatsheet.md` に委譲し、本書は**このルーティン（パワーデバイスCHT/Zth）で使う場面→ワークフロー→cheatsheet章→読み方**の適用索引に絞る。
> 作業例はパワーデバイス模擬（6領域、phase5-3/5-4構成）。

---

## 0. このステップの位置づけ

**8 = フィールド駆動の可視化（ブランチB）。検証[7]の視覚的裏付けと報告図の作成が主目的。**

```
[6]完走 ──► reconstructPar ──► [8]ここ（温度分布/放熱経路/層別プロファイル）
        └─► log ──► [9a]Zth（ブランチA、ParaView不要）
```

このルーティンでの重要な原則:

- **Zthはここでは出さない**（ブランチA[9a]がlogから出す）。ParaViewで温度の時間変化を見ることはできるが、Zth(t)・構造関数の定量抽出は[9]の役割。両者を混同しない。
- **固体のみのケースは流れ場が無い**（U, p_rgh不要）。可視化対象は基本 **T のみ**。流体ケースの流線・速度ベクトルはここでは出てこない。
- **reconstructParが前提**（並列計算の場合）。ブランチBは時刻フィールドを要するため、ブランチAと違い `reconstructPar -allRegions` が必要。

---

## 1. 前提

```bash
# 並列結果を統合（対象時刻 or 最終時刻）
reconstructPar -allRegions -latestTime    # 過渡を見るなら時刻範囲指定
                                          # 例: -allRegions -time '0:0.5'

# マルチリージョンは内蔵リーダで開く
paraFoam -builtin
```

開いたら: Pipeline Browser で `<ケース>.foam` 選択 → Mesh Regions 全領域チェック → Cell Arrays で `T` → Apply（cheatsheet §0, §9）。

### B-1. 領域分割の検算（視覚版）

splitMeshRegionsが正しく6領域に分かれたかを目で確認する（checkMeshのcellZone数[5]の視覚的裏取り）。

- 全領域チェック → Representation: **Surface With Edges**（cheatsheet §2）
- Extract Block で領域別に色分けすると積層が分かりやすい（cheatsheet §9）

読み方:
- [ ] 6領域すべてが存在し、Z方向に正しく積層（chip最上〜heatsink最下）
- [ ] 抜けた層・潰れた層がない（あれば[5]のcellZone/分割へ戻る）

---

### B-2. 定常温度分布（どこが熱いか）

中央を通る縦断面で温度場を見る。

- `<ケース>.foam` 選択 → Slice（cheatsheet §5）
  - Slice Type: Plane、Origin (0,0,0)、Normal (1,0,0) または (0,1,0)（XZ/YZ縦断面）
- Coloring: T、Cool to Warm（cheatsheet §2）
- 時間アニメするなら「全時刻通しての最小最大」でカラー固定（cheatsheet §2コツ）

読み方:
- [ ] chipが最高温（Tjの位置）、stackを下るほど低温、heatsink底≈300K
- [ ] 温度が**chip直下で急に落ちる**ならはんだ界面の効き（B-3で定量化）
- 検証[7]のTj・温度降下パターンの視覚的裏付けになる

---

### B-3. 縦断面プロファイル：界面の温度ジャンプ ★最重要

**接触抵抗（はんだ/TIM）の効きを直接見る。検証2の視覚版。**

chip中心を通る鉛直線で温度プロファイルを取る（cheatsheet §8 Plot Over Line）。

- 対象（全領域）選択 → Filters → Data Analysis → Plot Over Line
  - Point1 = (0, 0, 0.0006)    chip上面（Z=0.6mm）
  - Point2 = (0, 0, -0.00494)  heatsink底面（Z=−4.94mm）
  - Resolution: 高め（500〜1000）
- Apply → T を縦軸、arc_length（≈Z）を横軸

読み方（このルーティンの核心）:
```
温度プロファイルは層内でなだらかな勾配、界面で挙動が変わる:

 T ↑
   │＼            chip（急な勾配＝小断面で熱流束密度大）
   │ ＼___
   │     ↓ ★はんだ界面：温度が段差状にジャンプ（接触抵抗）
   │      ＼___
   │          ＼______  dbc/baseplate（緩い勾配＝大断面）
   │                 ↓ ★TIM界面：小さめのジャンプ
   │                  ＼___ heatsink
   └──────────────────────────► Z（下向き）
```

- [ ] **はんだ界面で大きな温度段差**（ΔT_solder）。これが面積換算でK/Wに効く支配項（検証2のはんだ1.2 K/W支配の視覚的証拠）
- [ ] TIM界面の段差は相対的に小さい（大面積のため）
- [ ] 段差が**ほぼ無い**なら界面抵抗が効いていない → kappaLayers桁ミスの疑い（4b B-1へ）
- [ ] 各層内は単調で滑らか（折れ・振動があればメッシュ/スキーム）

> ΔT_界面 と 熱流束Q から界面のRth[K/W]≈ΔT/Q を読めば、設定値（t/κ÷面積）と突き合わせできる（検証2の独立チェックの一つ）。

---

### B-4. 拡がり熱抵抗の可視化（検証3の視覚版）

小さなchipから大きなbaseplateへ熱が広がる様子を見る。

- Slice（縦断面）に Contour（等温面、cheatsheet §8）を重ねる、または
- 異なるZ高さで水平 Plot Over Line を複数取り、温度分布の横幅が下層ほど広がるのを見る

読み方:
- [ ] 等温線が下層ほど横に広がる「熱の円錐」→ 拡がり伝導を捉えている（検証3 ✓）
- [ ] 広がらず1Dのまま → 横メッシュ粗すぎ（4aへ）

---

### B-5. 界面熱流束分布（任意・詳細）

界面パッチ上の熱流束分布を見て、chip直下への集中と下層での分散を確認（cheatsheet §12-E）。

- Extract Block で界面を取り出す → 熱流束で着色（wallHeatFlux後処理フィールド or Calculator）

読み方:
- [ ] 上層界面ではchip直下に熱流束集中、下層界面では分散 → 拡がりの定量的傍証

---

### B-6. 過渡の可視化（任意）

温度場の時間発展。**Zth(t)の定量はここではなく[9a]**だが、過渡の様子を掴むのに有用。

- 時間アニメ（cheatsheet §3）でTの広がりを見る
- chip中心セルを選択 → Plot Selection Over Time（cheatsheet §8）でTj(t)
  - これはZth(t)の素（Zth=(Tj−300)/20）だが、定量CSV化は[9a]のlog抽出で行う

## 3. 報告用の標準図セット

検証と報告に最低限そろえる3枚:

1. **定常温度分布**（B-2: 縦断面Slice + T着色）— 全体像、Tj
2. **縦断面プロファイル**（B-3: Plot Over Line）— 界面の温度段差＝接触抵抗の効き ★
3. **拡がり可視化**（B-4: 等温面）— 3D伝導の証拠

> 画像は File → Save Screenshot（cheatsheet §10）。状態は .pvsm 保存で再現可（cheatsheet 付録）。

---

## 4. このルーティン特有の注意

- **Zthをここで出さない**: 温度の時間変化は見えるが、Zth(t)・構造関数の定量抽出はブランチA[9a/9b]。役割分担を守る（ParaViewを開かずともZthは取れる）。
- **固体のみは流れ無し**: U/p_rghは無い or アーティファクト。Tに集中。流線・Glyphの出番は流体ケースのみ。
- **reconstructPar必須**: ブランチBは時刻フィールドが要る。`-allRegions` を忘れると一部領域が読めない。
- **マルチリージョンは -builtin**: 通常のparaFoamで一部領域が見えないときは `paraFoam -builtin`（cheatsheet §11）。

## 5. 完了ゲート（→ ブランチA[9] / 文書化）

```
[8] 完了チェック
  □ reconstructPar -allRegions 済み / paraFoam -builtin で全領域表示
  □ B-1 領域分割を視覚確認（6領域・正しい積層）
  □ B-2 定常温度分布（Tj位置・降下パターン）
  □ B-3 縦断面プロファイルで界面の温度段差を確認 ★（検証2の裏付け）
  □ B-4 拡がりを確認（検証3の裏付け）
  □ 報告用3図を保存
```

ブランチBは検証[7]の視覚的裏付けと報告図で役割を終える。定量出力（Zth/構造関数）はブランチA[9]へ。

## 6. 他形状への転用（差分ポイント）

| 変わる前提 | 8で変わる箇所 |
|---|---|
| 流体領域が入る | 流線（StreamTracer）・速度ベクトル（Glyph）が主役に（cheatsheet §6,§7）。放熱経路＝対流ループの可視化 |
| 室内気流（定常） | 居住域のSlice温度分布、吹出し気流のStreamTracer、目的関数（温度ムラ）の可視化。Plot Over Lineは代表線で快適性評価 |
| 非定常パルス | 時間アニメが主。Plot Selection Over Timeで応答波形 |
| 界面抵抗が無いモデル | B-3の温度段差は出ない（連続プロファイル）。段差の有無で界面モデルの有無を逆引きできる |

**不変なもの**: 「可視化は検証の裏付けと報告が役割、定量はブランチAが担う」という分業、縦断面Plot Over Lineで界面の効きを読む手法、cheatsheetへの操作委譲。
