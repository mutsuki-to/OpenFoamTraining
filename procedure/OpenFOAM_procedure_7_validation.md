# 手順書 [7] 結果の妥当性検証

> 上位ルーティン `OpenFOAM_calc_routine.md` の §2[7]・§4 を展開した詳細手順書。
> 「計算が通った」と「物理的に正しい」は別。CFD結果を**予測値・実測値と定量照合**して妥当性を判定する。
> 作業例はパワーデバイス模擬（6領域、発熱20W、底面300K、phase5-3/5-4構成）。

---

## 0. このステップの位置づけ

**7 = 数値結果が物理的に正しいかを定量判定するステップ。** 6a/6bは「破綻していないか」の検査だったが、7は「物理が合っているか」の検証。

検証は3層に分かれ、**完成タイミングが違う**:

```
[6]完走
  ├─ 即検証（log/fieldだけで完結）        ─────► [7]本体
  │    検証1 エネルギー収支
  │    検証2 定常Zth vs 予測
  │    検証3 拡がり抵抗（CFD vs 1D）
  │
  └─ ブランチA[9a→9b]完了後に完成
       検証4 T3Ster実測との帯域照合  ◄──── [9b]構造関数が前提
```

> 本手順書は検証1〜3を本体とし、検証4は[9b]完了後に実施する前方参照として置く。ルーティン上、**最終検証の重い部分（実測照合）はブランチA完走後**に来る。

## 1. 検証1: エネルギー収支（発熱W = 底面熱流束）★最も基本

側面が断熱（zeroGradient）なので、定常では**発熱20Wの全量が底面（コールドプレート）から抜ける**。これが閉じなければ物理が破綻している。

### 定性チェック（まず無料でできる）

```bash
grep "Min/max T" log.chtMultiRegionFoam | tail -12
```
- chip の Tj が頭打ち（dTj/dt ≈ 0）＝定常近傍＝蓄熱速度≈0＝**Q_bottom ≈ Q_gen(20W)**
- Tj がまだ上昇中なら非定常。蓄熱分だけ Q_bottom < 20W（後述）

### 定量チェック（wallHeatFlux）

底面パッチの熱流束を積分して 20W と比較する。

```bash
# heatsink領域の最終時刻で壁面熱流束を後処理
chtMultiRegionFoam -postProcess -func wallHeatFlux -region heatsink -latestTime
# 出力ログ/ファイルで heatsink_bottom の積分値を確認
```

期待:
```
patch heatsink_bottom : ... integral ≈ −20 W   （符号は熱が抜ける向き）
```

> 版によって invocation が異なる場合は `postProcess -func wallHeatFlux -region heatsink -latestTime` を試す。要は**底面パッチ上の熱流束積分[W]**が取れればよい。直列経路なので各界面の積分もほぼ20Wになる（中間チェックに使える）。

### 判定と切り分け

| 観測 | 解釈 | 戻り先 |
|---|---|---|
| 定常で Q_bottom ≈ 20W（数%内） | エネルギー収支OK | — |
| Q_bottom < 20W だが Tj上昇中 | まだ非定常（蓄熱中）。正常 | endTime延長 or 待つ |
| 定常でも Q_bottom << 20W | 側面から熱漏れ（zeroGradientでない面がある）or 発熱過小 | 4b changeDict / Part0 fvOptions |
| Q_bottom > 20W | 物理的にありえない。符号/領域/積分対象の取り違え | 後処理コマンド見直し |

## 2. 検証2: 定常Zth vs 予測値

ステップ2で作った予測（物差し）とCFD結果を突き合わせる。

### CFD定常Zthの取得

```
Tj_final = chip の最終Max T（log最終の Min/max T、または field）
CFD定常Zth = (Tj_final − T_ref) / P     （T_ref=300K, P=20W）
```
```bash
grep "Min/max T" log.chtMultiRegionFoam | tail   # chip領域の最終max T
```

### 予測の積み上げ（手順書[4b]§B-1の面積換算に基づく）

```
予測定常Zth = CFD基準(界面なし) + Σ ΔR_界面

  CFD基準(界面なし, phase5-3)            = 0.323 K/W （拡がり込み実測値）
  ΔR_solder = Rth_solder / A_chip
            = 3.0e-5 / 25e-6   (chip 5×5mm) = 1.2   K/W  ← 小面積で支配的
  ΔR_TIM    = Rth_TIM / A_baseplate
            = 2.25e-4 / 3600e-6             = 0.063 K/W

  予測 ≈ 0.323 + 1.2 + 0.063 ≈ 1.586 K/W
```

**★予測は必ず「実際に使ったdictの値」で再計算する**。界面の `t/κ`（m²K/W）を dict から逆算し、接触面積で割って K/W にする。**材料Rth（cm²K/W）ではなく面積換算したK/Wで効き具合を見る**（材料RthはTIM>はんだだが、面積差144倍で逆転しはんだが支配的）。

### 判定と切り分け

| 観測 | 解釈 | 戻り先 |
|---|---|---|
| `|CFD − 予測|/予測` が数%内 | 整合。妥当 | — |
| CFDが予測の数倍大 | 界面 kappaLayers の桁ミスで界面が断熱壁化（静かで致命的） | 4b B-1（t/κ再検算） |
| CFDが予測より小 | 側面漏れ / 発熱過小 / 界面抵抗が効いていない | 4b / Part0 |
| **予測自体がCFDと大きく食い違う** | 予測の面積換算ミス（材料Rthのまま足した等）。**この不一致自体が界面設定の単位/面積ミスの検出器**になる | 予測の組み立て[2]/[4b]を見直す |

> 予測とCFDのズレは「どちらが正しいか」ではなく「設定の自己整合チェック」として使う。両者が独立に同じ値に収束して初めて妥当。

## 3. 検証3: 拡がり抵抗（CFD vs 1D）

CFDが3D（横方向拡散）を正しく捉えているかの確認。

### 1D積み上げとの比較（界面なしの基準で）

```
1D解析（厚さ方向のみ）:
  Rth_chip      = 0.0003/(148×25e-6)  ≈ 0.081 K/W
  Rth_ceramic   = 0.00064/(170×16e-4) ≈ 0.002 K/W
  Rth_baseplate = 0.003/(400×36e-4)   ≈ 0.002 K/W
  Σ(1D) ≈ 0.086 K/W

CFD基準(界面なし) ≈ 0.323 K/W
差（約4倍）= 拡がり熱抵抗
  chip断面25mm² → baseplate 3600mm² へ熱が広がる過程の抵抗
  CFDは自動で計算、1Dは捉えられない
```

### 判定

- [ ] CFD基準 が 1D和より**有意に大きい**（拡がり抵抗分）→ 3D物理を捉えている ✓
- [ ] CFD基準 ≈ 1D和（拡がりが出ない）→ **横方向拡散を解像できていない**（横メッシュ粗すぎ/形状エラー）→ 4a（refinement/XY解像度）

> 「定常到達」は数値的な見かけ。数学的には `Zth(t)=Zth_∞(1−Σaᵢexp(−t/τᵢ))` で漸近。baseplate横拡散τ≈L²/α=(0.03)²/1.2e-4≈7.5s の緩慢な裾は writePrecision以下で同値に見えるだけ。T3Sterの長時間測定でZthが緩く上昇し続ける挙動と同じ物理。

## 4. 検証4: T3Ster実測との帯域照合（→ [9b]完了後）

> この検証は**構造関数[9b]とT3Sterデータが前提**。[9a→9b]完了後に実施する。ここでは方法だけ前方参照で示す。

### 帯域ごとの対応

時間帯（または構造関数上の位置）が積層の各層に対応する。層ごとに照合する。

| 時間帯 | 対応する物理 | 照合のポイント |
|---|---|---|
| 〜数ms | chip + 直下界面（はんだ） | 立ち上がり。はんだ抵抗の効き |
| 5〜100ms | ceramic（材料感度study帯） | Al₂O₃/Si₃N₄/AlN で差が出る帯 |
| 0.1s〜 | baseplate横拡散 | 緩慢な裾、拡がり抵抗 |

### 照合の流れ（[9b]後）

```
1. CFD Zth(t)  と  T3Ster Zth(t) を対数時間で重ねる
2. 構造関数（累積 Cth–Rth）を重ね、ブレークポイントが層境界に対応するか
3. 帯域ごとに差を読む:
   - 形状差（実機の実効面積・取付け）
   - 界面抵抗の実測 vs 設定値
   - 接触条件（コールドプレート界面）
4. 差の物理的説明をつける（モデルの精度限界として記録）
```

- [ ] 構造関数のブレークポイントが既知の層境界に対応（[9b]ゲートと共通）
- [ ] 帯域ごとの差に物理的説明がついた

## 5. 物理的におかしいときの総合切り分け表

| 症状 | 第一に疑う | 戻り先 |
|---|---|---|
| 定常Zthが予測の数倍 | 界面 kappaLayers 桁ミス（断熱化） | 4b B-1 |
| 定常Zthが予測より小 | 側面熱漏れ / 発熱過小 | 4b changeDict / Part0 fvOptions |

## 7. 他形状への転用（差分ポイント）

| 変わる前提 | 7で変わる箇所 |
|---|---|
| 流体領域が入る | エネルギー収支に対流熱流束が加わる（壁の対流＋伝導）。出口が底面一点でなくなる |
| 定常解析（室内気流） | Zth照合は無し。検証は目的関数（居住域温度の標準偏差等）と物理直観／既知ケースとの比較に置換 |
| 実測がない | 検証4は省略。検証1〜3（収支・予測・拡がり）で閉じる |
| 界面抵抗が多数 | 検証2の予測積み上げを界面ごとに、面積換算で |

**不変なもの**: 「エネルギー収支で閉じるか」が最優先の物理チェック、予測とCFDの**独立した一致**を妥当性の根拠にする姿勢、CFDと1Dの差を拡がり抵抗で説明する習慣。
